## Agent Instructions
- **Use STATE.md as a state file, not a changelog** - It should only contain:
  - Current work in progress (if any)
  - Next steps to take
- **Keep STATE.md minimal** - Remove completed items, historical notes, and anything discoverable from the codebase. If it's over 50 lines, trim it.
- **Never add "completed" sections** - Completed work is visible in git history. STATE.md is for what's next, not what's done.
- **Do not write notes in STATE.md** - No gotchas, no patterns, no explanations. Only active work and next steps.
